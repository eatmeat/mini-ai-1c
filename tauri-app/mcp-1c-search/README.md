# 1c-search MCP Server

Высокопроизводительный MCP-сервер для поиска и навигации по исходным кодам конфигураций 1С:Предприятие (выгрузка в файлы).

## 🚀 Возможности

- **Мгновенный поиск символов**: Поиск определений процедур и функций по всей конфигурации за миллисекунды.
- **Интеллектуальный Grep**: Полнотекстовый поиск по `.bsl` и `.xml` файлам с поддержкой регулярных выражений и ограничением области поиска (`scope`).
- **Анализ метаданных**: Исследование структуры конфигурации (справочники, документы, реквизиты, формы) через парсинг `Configuration.xml` и `ConfigDumpInfo.xml`.
- **Поиск ссылок и анализ влияния**: Быстрое нахождение всех вхождений символа в коде.
- **Инкрементальный индекс**: Автоматическое обновление индекса только для измененных файлов на основе `mtime`.

## 🛠 Технологии

- **Rust**: Высокая производительность и безопасность памяти.
- **Tree-sitter**: Использование `tree-sitter-bsl` для точного извлечения AST и поиска заголовков методов.
- **SQLite**: Быстрое хранилище для индекса символов и метаданных.
- **Rayon**: Многопоточный парсинг файлов для максимального ускорения индексации.

## 📦 Установка и сборка

Сервер собирается как часть проекта Mini AI 1C, но может быть использован отдельно.

```bash
# Сборка
cargo build --release

# Исполняемый файл будет в target/release/mcp-1c-search.exe
```

## ⚙️ Настройка

Сервер настраивается через переменные окружения:

- `ONEC_CONFIG_PATH`: Путь к корневой директории с исходниками конфигурации 1С.

## 🔍 Доступные инструменты (Tools)

- `search_code`: Полнотекстовый поиск по коду.
- `find_symbol`: Поиск процедуры/функции в индексе.
- `get_symbol_context`: Получение полного кода метода.
- `get_file_context`: Чтение фрагмента файла вокруг строки.
- `list_objects`: Список объектов метаданных.
- `get_object_structure`: Полная структура конкретного объекта.
- `find_references`: Поиск всех упоминаний символа.
- `impact_analysis`: Анализ использования объекта в коде.
